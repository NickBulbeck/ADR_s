Architecture Decision Records

2018-12-05 Wordpress command-line interface (WP-CLI)

Decision

I will not attempt to install the WP-CLI on my laptop, because the space in my username is causing too many knock-on issues. The standard install contains scripts that are falling over at the space in /Users/user 1/. I might have edited the path to read /Users/'user 1' had I been able to find them (it worked for ~/.zshrc) but I was not able to find them. I tried circumventing the problem by creating a WordPress user, with a view to using this login only for WordPress work. Whilst the process of setting up another user, and updating ~/.bash_profile, was useful and instructive, and did allow the WP-CLI install to go further than it had previously, I reached a point where the npm package required me to chown a number of significant-looking folders relating to homebrew and npm. Doing this would risk compromising my main login and making me unable to do important NodeJS work on it.

Instead, I will attempt to install WP-CLI only on the mac, and will add my laptop at some point in future as and when I replace it.
